# /create-pr

**ultrathink**

現在のブランチの変更内容を分析し、Pull Request を作成してください。

## 手順

1. `git log` と `git diff` でブランチの変更内容を確認する
2. 変更内容を分析し、PR のタイトルと本文を生成する
3. Draft 状態で PR を作成する

## PR フォーマット

### タイトル
- 日本語で簡潔に変更内容を記載
- プレフィックス付き（feat:, fix: など）

### 本文
- 概要: 変更の目的と内容
- 変更点: 主な変更のリスト
- 確認事項: レビュアーに確認してほしいポイント

## 対象

$ARGUMENTS

（指定がない場合は現在のブランチ）
