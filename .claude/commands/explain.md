# /explain

**ultrathink**

指定されたコードや直前の実装について、詳細に説明してください。

## 説明内容

1. **何をしたか**: 変更・実装の概要
2. **なぜそうしたか**: 技術的判断の根拠
3. **どう動くか**: 処理フローの説明
4. **注意点**: 潜在的なリスクや制約

## 出力フォーマット

```markdown
## 概要
[変更内容の簡潔な説明]

## 技術的判断
[なぜこのアプローチを選んだか]

## 処理フロー
[ステップバイステップの説明]

## 注意点
[将来のメンテナンスで気をつけること]
```

## 対象

$ARGUMENTS

（指定がない場合は直前の実装内容）
