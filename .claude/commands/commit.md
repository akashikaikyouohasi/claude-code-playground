# /commit

**ultrathink**

現在の変更内容を分析し、Conventional Commits に従ったコミットメッセージを生成してコミットしてください。

## 手順

1. `git status` と `git diff` で変更内容を確認する
2. 変更内容を分析し、適切なコミットメッセージを生成する
3. ユーザーに確認を取ってからコミットを実行する

## コミットメッセージ規約

- プレフィックス: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`
- タイトル: 50文字以内、日本語で簡潔に
- 本文: 必要に応じて「なぜ」この変更をしたかを記載

## 対象

$ARGUMENTS

（指定がない場合は全ての変更をステージング）
